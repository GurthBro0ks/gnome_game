# Event Content Templates

## Purpose

This file defines the reusable content architecture for the weekly live-service cycle in **Gnome Survivor**. It is meant to prevent the event layer from becoming a pile of one-off ideas. Instead, each event week should be made from repeatable content bricks, scalable reward ladders, and expandable content pools.

This document covers:
- **Lucky Draw Week**
- **Treasure Week**
- **Echo Week**
- **Festival Ledger**
- their side loops, scoring ladders, shops, cadence, and scaling logic

It is designed for a game that may eventually support **100+ recurring NPCs, event items, treasure pools, clue chains, and layered difficulty growth** over a multi-year content arc.

---

## Core Event Philosophy

### 1. Events are repeatable frameworks, not handcrafted exceptions
Each event week should be built from the same base skeleton every cycle:
- earn event currency
- spend event currency
- score progress
- unlock milestone rewards
- engage one side loop
- interact with one weekly shop/exchange
- feed Festival Ledger meta-progress

### 2. Hoarding is intentional gameplay
Players should feel smart for planning ahead.
The event cycle should reward:
- discipline
- long-view saving
- route planning
- event targeting
- account specialization

### 3. Every event should support both short-term excitement and long-term account planning
A good week should answer both:
- “What do I do right now?”
- “What am I building toward over the next several cycles?”

### 4. Difficulty should compound through content layering, not raw stat inflation alone
Like a long-running roaming-visitor system, events should get denser over time by adding:
- new reward pools
- new side-loop variants
- new shop items
- new ledger branches
- new encounter hooks
- more complex decision pressure

---

## Global Weekly Event Structure

## 3-Week Cycle

1. **Lucky Draw Week**
2. **Treasure Week**
3. **Echo Week**

This repeats forever.

Above all three sits:
- **Festival Ledger** = meta reward track across the full cycle

Each week includes:
- main event currency
- main scoring ladder
- one side loop
- one themed shop or exchange
- one optional Post / Wanderer / Buried Clue tie-in
- Festival Marks gain

---

## Shared Event Content Bricks

Every event should be assembled from the same content-brick categories.

### Brick A — Currency Sources
Ways the player earns the week’s currency.

Examples:
- Daily Duties
- Strata progression
- Burrow Gather actions
- Burrow Posts
- Wanderer trades
- Crack Market purchases
- special login chain
- Festival Ledger milestones

### Brick B — Main Spend Action
What the player actually does with the event currency.

Examples:
- roll at Lucky Wheel
- target a Treasure Finder pool
- present Echoes to the Ancestral Depictor

### Brick C — Score Ladder
A set of milestone thresholds that reward planned spending.

Every event should have:
- accessible threshold for disciplined F2P players
- mid threshold for optimized players
- stretch threshold for heavy planners and spenders
- capstone tier that feels meaningful but not mandatory

### Brick D — Side Loop
The event’s secondary activity.
This should deepen the week without replacing its main purpose.

Examples:
- vendor chain
- bonus scoring board
- rotating shop
- reroll system
- clue chase
- special Wanderer subclass

### Brick E — Event Shop / Exchange
A targeted secondary sink for event participation.

Examples:
- Lucky Stall
- Treasure Exchange
- Echo Watch shop

### Brick F — Meta Feed
Every event should contribute to Festival Ledger.
This makes even a weak personal week still feel meaningful.

### Brick G — Story Tie-in
Each event should have at least one light story-facing component:
- Burrow Post chain
- Wanderer group
- Runaway rumor
- Smoke Whisper variant
- Buried Clue fragment

---

## Event Growth Model

To support years of content, each weekly event needs **expandable content layers**.

### Layer 1 — Launch-safe core
- one main currency
- one spend action
- one ladder
- one side loop
- one shop

### Layer 2 — Seasonal variants
- alternate reward pools
- region-themed item sets
- new Wanderers tied to the week
- temporary modifiers

### Layer 3 — Deep account hooks
- Strain-specific week bonuses
- Confidant tie-ins
- Burrowfolk side scoring
- Deepening bonus conditions

### Layer 4 — Legacy density
- old event reward pools remain partially relevant
- old shop items return through rotation
- prior-week fragments feed future systems
- event cast grows over time instead of resetting each season

This is how the event layer can grow to large content counts without collapsing into nonsense.

# Lucky Draw Week

## Core fantasy
A noisy, high-variance, reward-heavy week centered on pulling, luck, side vendors, and streak excitement.

## Main currency
- **Lucky Draws**

## Main spend action
- spend Lucky Draws at the **Lucky Wheel**

## Main player feelings
- anticipation
- short-burst excitement
- pity management
- planning when to stop or push further

## Core loop
1. earn Lucky Draws
2. roll Lucky Wheel
3. gain rewards and score
4. convert extras into Lucky Stall value
5. push ladder thresholds
6. earn Festival Marks

## Core reward categories
- Treasures
- Treasure Shards
- Polishes
- Clothes materials
- Confidant materials
- Burrowfolk materials
- Echo support items
- Festival Marks

## Side loop
### Lucky Stall
A weekly vendor that converts participation into a more controlled second reward path.

Lucky Stall should rotate:
- one always-useful account item
- one targeted growth item
- one event-exclusive cosmetic or vanity item
- one long-tail accumulation item

## Optional secondary mechanic ideas
- streak bonus after X rolls
- lucky-chain branch
- reroll token
- wheel hotspot bonus
- capped bonus pull on threshold hit

## Score ladder structure
Each cycle should have five ladder bands:
1. Entry
2. Disciplined
3. Optimized
4. Push tier
5. Capstone

The point is not exact numbers here.
The point is predictable psychological pacing.

## Long-term scaling ideas
- add themed wheel pools
- add rare Wanderer jackpot modifiers
- add Strata-specific reward tags
- add special Lucky Wheel skins/effects
- add historical “returned pools” for old content

# Treasure Week

## Core fantasy
A focused planning week where the player aims at desired growth lanes instead of relying on raw randomness.

## Main currency
- **Treasure Tickets**

## Main spend action
- target Treasure pools with **Treasure Finder**

## Main player feelings
- precision
- control
- research-driven planning
- efficient account routing

## Core loop
1. earn Treasure Tickets
2. choose target board/family/focus
3. spend through Treasure Finder
4. build score ladder progress
5. convert extras in Treasure Exchange
6. feed Festival Ledger

## Main value proposition
This is the “I know what my account needs” week.

## Treasure Finder targeting layers
Recommended targeting hierarchy:
1. board type
2. family or theme
3. role focus
4. optional event modifier

Possible board targets:
- Vault of Treasures
- Pathwheel
- Ancestral Depictor
- Provision Board / Burrow production

## Side loop
### Treasure Exchange
Lets players convert excess or lower-priority event value into more useful materials.

Treasure Exchange should favor:
- targeted account correction
- materials that fix bottlenecks
- event leftovers turning into future value

## Optional secondary mechanic ideas
- rotating treasure families
- featured “old kingdom” set
- Confidant-linked treasure set
- Buried Clue treasure chain
- account-needs recommendation tag in UI later

## Score ladder structure
Treasure Week should feel a little less explosive than Lucky Draw Week, but more satisfying for planners.

Reward ladder emphasis:
- targeted Treasure materials
- Shards
- Polishes
- Treasure-specific side materials
- Strata Seals fragments at higher bands later

## Long-term scaling ideas
- add more treasure families without replacing old ones
- add board-specific bonus tracks
- add weekly advisory modifiers
- add “memory sets” tied to later Deepenings
- add premium-but-not-exclusive convenience targeting

# Echo Week

## Core fantasy
A ritual, accumulation, and offering week centered on time conversion, Echo management, and shrine pacing.

## Main currencies / spend objects
- **Echoes**
- **Echo Rushes**
- **Elder Books**

## Main spend action
- present Echoes through the **Ancestral Depictor**

## Main player feelings
- buildup
- ritual rhythm
- ceremonial payoff
- long-hoard satisfaction

## Core loop
1. collect Echoes from normal play and saved stock
2. use Echo Rushes when timing matters
3. present Echoes at Ancestral Depictor
4. score progress into Echo Week ladder
5. gain Echo Shards
6. spend through Echo Watch
7. advance Festival Ledger

## Main reward categories
- Echo Shards
- Elder Books components
- Confidant trust items
- ritual materials
- late-ladder Treasure support
- rare story hooks

## Side loop
### Echo Watch
A ritual-side scoring and reward structure.
It should feel like observing the consequences of offering, not just another shop.

Echo Watch may include:
- round scoring
- ritual observations
- omen tracks
- hidden thresholds
- special Smoke Whispers during event windows

## Optional secondary mechanic ideas
- omen board
- offering chain bonus
- ritual harmony states
- Wanderer pilgrim surge
- Hushline-themed hidden offer conditions

## Score ladder structure
Echo Week should feel slightly more ceremonial and cumulative than the other two weeks.
The pacing should create a sense of planned release.

## Long-term scaling ideas
- more ritual encounter chains
- Deepening-linked omen systems
- memory-themed alternate offerings
- rotating Elder Book families
- late-game Hushline and Memory Shift cross-hooks

# Festival Ledger

## Core fantasy
The meta event that makes all three weeks feel connected.

## Main currency
- **Festival Marks**

## Function
Festival Ledger provides:
- cross-week continuity
- forgiveness for imperfect weeks
- longer reward cadence
- stronger retention across the whole cycle
- a reason to engage with all three weekly systems

## Reward track structure
Recommended lanes:
- free lane
- optional upgraded lane later
- themed mini milestones
- occasional cycle capstones

## Good reward types
- universal materials
- event correction items
- rare but not mandatory cosmetics
- long-tail collection materials
- Confidant support
- Burrowfolk support
- Treasure support
- Strata Seals fragments later

## Scaling logic
Festival Ledger should get denser over time by adding:
- more branch rewards
- more cycle themes
- cycle-specific story posts
- returning historical tracks
- cross-week challenge conditions

## Difficulty and Content Compounding Model

Because this game may eventually support a very large cast and long event history, event difficulty should compound through **density tiers**.

## Tier 1 — Base comprehension
The player understands the week.
- one obvious goal
- one side loop
- one reward ladder

## Tier 2 — Choice pressure
The player chooses where to aim.
- multiple targets
- shop prioritization
- side loop tradeoffs

## Tier 3 — Cross-system optimization
The player combines weeks with account strategy.
- Strain synergy
- Confidant synergy
- Burrow production timing
- Crack Market timing
- Post timing

## Tier 4 — Legacy mastery
The player recognizes:
- old recurring Wanderers
- old event pools
- recurring ledger structures
- hidden rotation logic
- long-haul reward planning

This lets the game feel deeper over time without requiring incomprehensible frontloaded complexity.

## Production Rules

1. Do not invent a new event framework when an existing one can carry the idea.
2. Add density through variants, not through chaos.
3. Every event must reward both engagement and planning.
4. No weekly event should require knowledge of every old event to be playable.
5. Old event content should remain partially relevant, not permanently dominant.
6. Festival Ledger should make weak weeks feel recoverable.
7. Event shops must fix frustration, not multiply it.
8. Event story hooks should deepen the world without blocking normal play.
